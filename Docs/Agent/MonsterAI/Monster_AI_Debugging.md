# 怪物 AI 调试工具说明

最后更新：2026-05-10

## 1. 调试目标

每个怪物 AI 行为都必须能被观察和复盘。调试信息至少覆盖当前状态、当前目标、当前招式、评分结果、黑板值、EQS 结果和失败原因。

## 2. 日志分类

| 分类 | 用途 |
| --- | --- |
| `LogMonsterAI` | 状态切换、目标选择、阶段变化和调试快照。 |
| `LogMonsterCombat` | 招式评分、招式选择、冷却和失败原因。 |
| `LogMonsterPart` | 部位受伤、破坏、硬直累计和倒地触发。 |

## 3. 调试快照

`FMonsterAIDebugSnapshot` 保存最近一次关键决策：

- 当前状态。
- 当前目标。
- 当前招式。
- 最近评分结果数组。
- 黑板摘要。
- EQS 摘要。
- 最近失败原因。
- 状态标签和阶段标签。

## 4. 建议屏幕调试字段

| 字段 | 示例 |
| --- | --- |
| 状态 | `Attack`、`Approach`、`Enraged` |
| 目标 | 玩家角色名称 |
| 招式 | `Bite_Left`、`Tail_Sweep` |
| 失败原因 | `距离过远`、`冷却未结束` |
| 黑板摘要 | `距离=820 角度=24 玩家数=2` |
| EQS 摘要 | `接近点=有效 得分=0.72` |

## 5. 验证要求

- 编译通过只能证明 C++ 结构有效，不能证明怪物战斗行为正确。
- 编辑器运行时测试必须观察日志是否出现评分、失败回退和状态切换。
- 每次测试结果写入 `Monster_AI_Context_Memory.md` 的临时调试信息区域。
