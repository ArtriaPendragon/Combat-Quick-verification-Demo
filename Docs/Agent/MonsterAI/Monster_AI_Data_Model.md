# 怪物 AI 数据结构设计

最后更新：2026-05-10

## 1. 数据建模原则

怪物 AI 的可调参数必须进入数据资产或数据表。C++ 只提供通用评分、校验和状态承载逻辑，不把某只怪物的固定招式、距离或阶段条件写死。

## 2. 数据资产

`UMonsterAIDataAsset` 是怪物 AI 的主配置资产。一个怪物蓝图可以引用一个数据资产，也可以由同类怪物共享同一份资产。

| 字段 | 用途 |
| --- | --- |
| 怪物显示名称 | 调试和编辑器识别。 |
| 初始状态 | 生成后默认进入的怪物状态。 |
| 初始阶段标签 | 例如普通阶段、愤怒阶段前置标签。 |
| 最大生命值 | 用于阶段切换和濒死判断。 |
| 愤怒生命比例 | 当前生命低于该比例时可进入愤怒候选。 |
| 濒死生命比例 | 当前生命低于该比例时可进入濒死候选。 |
| 感知半径 | 行为树或感知系统搜索玩家的默认范围。 |
| 仇恨切换阈值 | 新目标仇恨超过当前目标多少时允许切换。 |
| 招式表 | 所有可评分招式。 |
| 部位表 | 所有可破坏或可触发硬直的部位。 |

## 3. 招式数据

每个招式使用 `FMonsterAttackMoveData` 描述。

| 字段 | 用途 |
| --- | --- |
| 招式编号 | 行为树、动画和日志使用的稳定标识。 |
| 招式标签 | 用 `GameplayTag` 描述招式类型。 |
| 显示名称 | 编辑器和调试显示。 |
| 最小/最大距离 | 约束招式可命中距离。 |
| 最小/最大角度 | 约束目标相对怪物朝向的角度范围。 |
| 冷却时间 | 防止招式连续重复。 |
| 基础权重 | 作为评分基础。 |
| 距离权重 | 控制距离命中度对评分的影响。 |
| 角度权重 | 控制朝向匹配度对评分的影响。 |
| 仇恨权重 | 允许高仇恨目标提高招式分。 |
| 多目标奖励 | 周围玩家越多时提升范围招式评分。 |
| 愤怒奖励 | 愤怒阶段额外分。 |
| 需要/阻止状态标签 | 控制硬直、倒地、吼叫等状态下能否使用。 |
| 需要/阻止阶段标签 | 控制阶段限定招式。 |
| 硬直阻止 | 当前硬直时是否禁止该招式。 |

## 4. 部位数据

每个部位使用 `FMonsterPartData` 描述。

| 字段 | 用途 |
| --- | --- |
| 部位编号 | 伤害结算、日志和蓝图查询使用。 |
| 显示名称 | 编辑器和调试显示。 |
| 最大耐久 | 部位完整时的耐久。 |
| 破坏阈值 | 耐久低于该值时标记破坏。 |
| 硬直阈值 | 累计伤害达到该值时触发硬直。 |
| 是否可破坏 | 控制是否输出部位破坏事件。 |
| 部位标签 | 例如头、尾、翼、弱点等语义。 |

## 5. 运行时评分上下文

`FMonsterAttackScoreContext` 由行为树、黑板或组件组装，包含：

- 当前目标距离。
- 当前目标相对角度。
- 当前玩家数量。
- 当前目标仇恨值。
- 当前是否处于硬直。
- 当前状态标签。
- 当前阶段标签。
- 当前世界时间。

## 6. 评分结果

`FMonsterAttackScoreResult` 保存一次招式评估结果：

- 招式编号。
- 总分。
- 是否可执行。
- 失败原因。
- 距离分、角度分、权重分、仇恨分、多人分、阶段分。

这些结果必须保留给调试面板、日志和上下文记忆。
