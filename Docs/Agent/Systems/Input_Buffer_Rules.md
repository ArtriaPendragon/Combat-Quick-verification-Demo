# 输入缓冲与怪物招式回退规则

最后更新：2026-05-10

## 1. 当前定位

历史玩家输入缓冲仍保留在 `UCombatAgentCombatComponent` 中，但当前 AI Agent 标准已经切换为怪物 AI。怪物 AI 不使用玩家输入缓冲作为核心机制，而使用招式评分、可执行检查和失败回退。

## 2. 怪物招式候选规则

每次攻击决策由行为树或黑板组装 `FMonsterAttackScoreContext`，再交给 `UMonsterCombatComponent` 评分。

评分输入包括：

- 当前目标。
- 目标距离。
- 目标角度。
- 当前玩家数量。
- 当前目标仇恨值。
- 当前硬直状态。
- 当前状态标签。
- 当前阶段标签。
- 当前世界时间。

## 3. 失败回退规则

| 失败原因 | 回退行为 |
| --- | --- |
| 距离过远 | 接近目标或选择接近点。 |
| 距离过近 | 后撤、侧移或等待。 |
| 角度不匹配 | 转向目标。 |
| 冷却未结束 | 等待短时间或重新评分。 |
| 状态标签阻止 | 进入硬直、倒地或恢复分支。 |
| 阶段标签不满足 | 回到普通招式评分。 |
| 目标无效 | 搜索目标或巡逻。 |
| 数据资产缺失 | 停止攻击分支并记录配置错误。 |

## 4. 必要日志

- 招式评分数组。
- 最高分招式编号。
- 是否可执行。
- 失败原因。
- 回退移动意图。
