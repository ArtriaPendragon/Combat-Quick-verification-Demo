# 怪物行为优先级矩阵

最后更新：2026-05-10

## 1. 高层优先级

| 优先级 | 状态 | 说明 |
| --- | --- | --- |
| 1 | 死亡 | 停止所有战斗决策。 |
| 2 | 倒地或硬直 | 暂停招式选择，等待恢复或动画事件。 |
| 3 | 换区或濒死 | 进入换区、逃离或保守行为。 |
| 4 | 愤怒切换 | 可触发吼叫、阶段标签更新和招式权重变化。 |
| 5 | 当前目标有效 | 进入战斗循环。 |
| 6 | 发现目标 | 更新仇恨并选择目标。 |
| 7 | 无目标 | 巡逻或等待。 |

## 2. 战斗循环优先级

| 条件 | 行为 |
| --- | --- |
| 目标距离在招式范围内且角度匹配 | 请求招式评分。 |
| 最高分招式可执行 | 执行招式。 |
| 目标距离过远 | 接近目标。 |
| 目标距离过近 | 后撤或等待。 |
| 目标角度不匹配 | 转向目标。 |
| 招式冷却未结束 | 等待或重新评分。 |
| 目标无效 | 清空目标并回到搜索。 |

## 3. 禁止事项

- 不在行为树任务中写大量招式专属条件。
- 不在 `AIController` 或 `Character Tick` 中堆叠怪物行为。
- 不绕过数据资产直接写死距离、角度、冷却和权重。
