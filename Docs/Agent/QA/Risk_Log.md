# 风险记录

最后更新：2026-05-10

## RISK-001 规则与实现漂移

- 描述：文档标准和 C++ 实现可能在后续迭代中脱节。
- 影响范围：怪物 AI 全部模块。
- 严重度：中。
- 缓解方式：每次实现后更新当前状态、变更记录、迭代记录和上下文记忆。
- 当前状态：开放。

## RISK-002 运行时资产缺失

- 描述：当前只有 C++ 骨架，没有怪物数据资产、行为树、黑板和 EQS 资产。
- 影响范围：怪物 AI 运行时行为。
- 严重度：高。
- 缓解方式：下一轮优先创建资产实例并进行编辑器播放测试。
- 当前状态：开放。

## RISK-003 评分系统尚未接入动画执行

- 描述：`UMonsterCombatComponent` 可以选择招式，但尚未驱动蒙太奇、命中盒或伤害。
- 影响范围：战斗表现和真实可玩性。
- 严重度：中。
- 缓解方式：先由行为树写黑板和日志，再逐步接入动画。
- 当前状态：开放。

## RISK-004 部位伤害没有真实碰撞来源

- 描述：部位组件可以结算部位伤害，但没有接入武器命中、碰撞体或伤害事件。
- 影响范围：部位破坏和硬直验证。
- 严重度：中。
- 缓解方式：下一步提供蓝图或 C++ 伤害入口测试用例。
- 当前状态：开放。

## RISK-005 Live Coding 影响外部构建

- 描述：编辑器开启 Live Coding 时，普通外部构建会被 UBT 阻止。
- 影响范围：命令行验证。
- 严重度：低。
- 缓解方式：本轮使用 `-NoHotReloadFromIDE` 完成构建；若链接失败，需要关闭编辑器或使用编辑器内 Live Coding。
- 当前状态：已缓解。

## RISK-006 历史玩家系统与怪物 AI 混淆

- 描述：项目中仍保留玩家输入缓冲和玩家锁定目标代码，可能与怪物 AI 目标混淆。
- 影响范围：文档理解和后续开发优先级。
- 严重度：中。
- 缓解方式：文档中明确历史玩家系统不是当前怪物 AI 主标准。
- 当前状态：已缓解。

## RISK-007 目录迁移后的路径缓存

- 描述：源码和文档移动后，IDE 或 UE 中间文件可能暂时保留旧路径缓存。
- 影响范围：Rider、Visual Studio、UHT 中间文件和开发者导航。
- 严重度：低。
- 缓解方式：本轮已完成命令行编译验证；如 IDE 未刷新，需要重新加载项目或重新生成项目文件。
- 当前状态：已缓解。

## RISK-008 行为树黑板键未配置

- 描述：C++ 行为树任务已经存在，但编辑器行为树资产必须正确配置黑板键，否则任务会读写空键或无效数据。
- 影响范围：招式选择、移动回退和调试快照。
- 严重度：中。
- 缓解方式：已新增 `MonsterBlackboardKeys.h` 并给任务和服务设置默认键名；下一轮创建资产时仍需创建同名键并验证。
- 当前状态：部分缓解。

## RISK-009 默认行为树未配置

- 描述：`AMonsterAIController` 已能运行默认行为树，但控制器实例或蓝图必须配置 `DefaultBehaviorTree`。
- 影响范围：怪物运行时 AI 是否启动。
- 严重度：中。
- 缓解方式：下一轮创建控制器蓝图并配置行为树资产；未配置时日志会输出失败原因。
- 当前状态：开放。
